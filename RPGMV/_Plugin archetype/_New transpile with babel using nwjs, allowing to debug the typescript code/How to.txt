When using NW.js (previously known as Node-Webkit), you can still debug your TypeScript code by following these steps. Here's how you can debug TypeScript code in NW.js using Visual Studio Code:

1- Set up your NW.js project:

-Create a new NW.js project or use an existing one.
-Configure the necessary files and directories for your project, including your TypeScript source files.

2- Install dependencies:

-Make sure you have Node.js, TypeScript, and Babel installed globally or locally in your project.
(ME: NodeJS should be already, but test locally without and with it)

3- Configure your 'tsconfig.json' file:

-Set up the 'tsconfig.json' file in your project to include the necessary TypeScript options. Ensure that the 'sourceMap' option is set to true to generate source maps during compilation.

4- Configure Babel for NW.js:

-Set up Babel to transpile your TypeScript code to JavaScript for NW.js.
-Configure Babel to preserve the source maps generated by TypeScript.

3.1 & 4.1 - Further explanation of these points
In point 4, where we configure Babel for NW.js, we need to ensure that Babel is set up to transpile our TypeScript code to JavaScript while preserving the source maps generated by TypeScript.

To achieve this, follow these steps:

  1- Install the necessary Babel packages:

   -Install the required Babel packages for transpiling TypeScript code. You will need the '@babel/preset-env', '@babel/preset-typescript', and '@babel/preset-stage-3' presets, along with any other Babel plugins you may need for your project.

  2- Create a Babel configuration file:

   -Create a new file named .babelrc in the root directory of your project.
   -Configure Babel to use the necessary presets and plugins. Here's an example configuration:

--------------------------------------------------------------------------------------------------
{
  "presets": [
    ["@babel/preset-env", {
      "targets": {
        "node": "current"
      }
    }],
    "@babel/preset-typescript",
    "@babel/preset-stage-3"
  ],
  "sourceMaps": true
}
--------------------------------------------------------------------------------------------------

  -In this example, we use the '@babel/preset-env' preset with the ' "node": "current" ' target to ensure compatibility with the current version of NW.js.
  -The '@babel/preset-typescript' preset enables Babel to transpile TypeScript syntax.
  -The '@babel/preset-stage-3' preset includes additional features and proposals.
  -Setting ' "sourceMaps": true ' ensures that Babel preserves the source maps generated by TypeScript during transpilation.

  3- Configure your TypeScript compiler to use Babel:

  -In your 'tsconfig.json' file, add the following options under ' "compilerOptions": '
--------------------------------------------------------------------------------------------------
{
  "compilerOptions": {
    // ...
    "outDir": "dist",
    "module": "commonjs",
    "esModuleInterop": true,
    "allowJs": true,
    "skipLibCheck": true,
    "resolveJsonModule": true,
    "noEmit": false,
    "plugins": [
      { "transform": "@zerollup/ts-transform-paths" },
      { "transform": "babel-plugin-transform-typescript-metadata" }
    ]
  }
}
--------------------------------------------------------------------------------------------------

  -The ' "allowJs": true ' option allows TypeScript to compile JavaScript files as well, which is useful when using Babel.
  -The "plugins" array includes any necessary TypeScript plugins. In this example, we have added two plugins, @zerollup/ts-transform-paths and babel-plugin-transform-typescript-metadata, which may be required in certain projects.

  By following these steps, Babel will be configured to transpile your TypeScript code to JavaScript while preserving the source maps generated by TypeScript. This ensures that your debugging experience in Visual Studio Code maps back to the original TypeScript files, even though they are transpiled using Babel for use with NW.js.

- Open the '.vscode/launch.json' file in your project (create one if it doesn't exist).
- Create a new launch configuration or modify an existing one to debug NW.js with TypeScript. Here's an example configuration:
--------------------------------------------------------------------------------------------------
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Debug NW.js TypeScript",
      "runtimeExecutable": "${workspaceFolder}/path/to/nw",
      "program": "${workspaceFolder}/path/to/your/app/index.html",
      "runtimeArgs": ["--remote-debugging-port=9222"],
      "sourceMaps": true,
      "outFiles": ["${workspaceFolder}/dist/**/*.js"],
      "preLaunchTask": "build" // Optional: Add a task to build your TypeScript files before debugging
    }
  ]
}
--------------------------------------------------------------------------------------------------
-Update the "runtimeExecutable" path to point to the 'nw' executable or the NW.js binary appropriate for your operating system.

-Update the "program" path to point to the entry HTML file of your NW.js application.

-Ensure the "runtimeArgs" include the '--remote-debugging-port' option to enable remote debugging.

7- Debugging with TypeScript:

-Open your project in Visual Studio Code and set breakpoints in your original TypeScript files.
-When the breakpoints are hit, the debugging experience will be provided by the Chrome or Edge DevTools, which support TypeScript debugging.
-You can inspect variables, step through code, and navigate the TypeScript source just like in regular web debugging.
